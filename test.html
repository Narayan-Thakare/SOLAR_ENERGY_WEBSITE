Interface

  • 1. What is an interface?
    Answer

    Interface is java member ,An interface in Java is used to achieve pure abstraction. Concrete non-static methods are not allowed in an interface.

  • 2. Why do we use an interface?
    Answer

    1. To achieve multiple inheritance.

    2. To enforce a contract for implementing classes.

  • 3. What can we create inside an interface?
    Answer

    - Public non-static abstract methods.
    - Public static final variables.
    - Default methods.
    - Static methods.
    - Private methods (since Java 9).
    - Private static methods (since Java 9).
    - Nested types (interfaces, classes, enums).
    - Constant declarations.
    - Public methods (by default).
    - Abstract methods (implicitly public).

  • 4. List 10 things we cannot create inside an interface?
    Answer

    - Constructors
    - Instance fields (non-static fields)
    - Non-public methods (e.g., protected methods)
    - Non-final variables
    - Static blocks
    - Instance blocks
    - Private constructors
    - Non-abstract methods (without default or static keywords)
    - Multiple inheritance (in terms of classes)
    - Static initializers

  • 5. Why can we not create a constructor inside an interface?
    Answer

    We cannot create a constructor inside an interface because interfaces cannot be instantiated. Constructors are meant for initializing objects, which interfaces do not do. Classes that implement interfaces provide their own constructors.

    • 6. Why can we not create a static block inside an interface?
      Answer

      We cannot create a static block in an interface because interfaces do not allow initialization code. They are meant to define method signatures, not implement logic.

    • 7. Can any member inside an interface be private by default?
      Answer

      No, it can only be public by default.

    • 8. Why are concrete methods not allowed in an interface?
      Answer

      Concrete methods are not allowed in interfaces to ensure that classes define their own method implementations.

    • 9. Are non-static methods by default public and abstract or final in an interface?
      Answer

      No, they are public and abstract by default, not final.

    • 10. We want to achieve inheritance between two classes, which keyword can we use?
      Answer

      extends

    • 11. What is a marker interface?
      Answer

      A marker interface is an interface with no methods or fields. It is used to signal to the JVM or compiler that the class implementing it should be treated differently (e.g., `Serializable` or `Cloneable`).

    • 12. What is the difference between abstract classes and interfaces?
      Answer

      1. Abstract classes can have both abstract and non-abstract methods.
      2. Interfaces could only have abstract methods until Java 8 allowed default and static methods.

    • 13. Is there a parent class of an interface?
      Answer

      No.

    • 14. How can we use multiple interfaces in a class?
      Answer

      1. A class can implement multiple interfaces.
      2. This allows a class to inherit behavior from multiple sources.

    • 15. Can an interface inherit a class?
      Answer

      No.

    • 16. Can we create an object of an interface?
      Answer

      No.

Exception And Exception Handling

  • 1. What is an exception?
    Answer

    An exception is a problem that occurs during the execution of software due to an abnormal situation.

  • 2. Types of exceptions
    Answer

    1. Checked exception
    2. Unchecked exception

  • 3. What is a checked exception?
    Answer

    Exceptions that the compiler checks during compilation are called checked exceptions.

  • 4. What is an unchecked exception?
    Answer

    Exceptions that the compiler does not check during compilation are called unchecked exceptions.

  • 5. Types of exceptions in Java
    Answer

    1. ArithmeticException
    2. ClassCastException
    3. NullPointerException
    4. ArrayIndexOutOfBoundsException
    5. IllegalArgumentException
    6. NumberFormatException
    7. IOException

  • 6. What is the difference between exception and error?
    Answer

    Exceptions are recoverable issues, while errors are generally unrecoverable issues related to system resources.

  • 7. Which classes belong to exceptions?
    Answer

    All exceptions are subclasses of the Throwable class.

  • 8. What is the parent class of Throwable?
    Answer

    The Object class is the parent class of Throwable.
